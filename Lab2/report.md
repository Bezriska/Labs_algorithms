## Теоретическая часть

### Сортировка Шелла (Shell Sort)

**Сортировка Шелла** — это улучшенная версия сортировки вставками, которая позволяет обменивать элементы на большом расстоянии друг от друга.

**Принцип работы:**
- Массив делится на подмассивы с шагом h
- Каждый подмассив сортируется
- Шаг постепенно уменьшается (h = h/2)
- При h = 1 выполняется обычная сортировка вставками

**Сложность:**
- Средний случай: O(n log² n)
- Худший случай: O(n²)
- Лучший случай: O(n log n)

### Бинарный поиск (Binary Search)

**Бинарный поиск** — алгоритм поиска элемента в отсортированном массиве путем многократного деления пополам.

**Принцип работы:**
- Сравнение искомого элемента с элементом в середине массива
- Если элемент меньше среднего — поиск в левой половине
- Если элемент больше среднего — поиск в правой половине
- Процесс повторяется до нахождения элемента или исчерпания массива

**Сложность:** O(log n)

## Описание реализации

### Структура данных

```c
typedef struct {
```

### Реализованные функции

#### 1. `initiate_data(char* data, double key)`
Создает и инициализирует структуру Data.

**Алгоритм:**
- Выделяет память для структуры
- Копирует строку в поле data
- Устанавливает значение ключа
- Возвращает указатель на созданную структуру


#### 2. `read(char* path)`
Читает данные из файла и создает массив структур.

**Алгоритм:**
- Открывает файл
- Читает количество элементов из первой строки
- Выделяет память для массива указателей
- Для каждого элемента:
  - Читает ключ
  - Читает строковые данные
  - Создает структуру Data
  - Добавляет в массив
- Закрывает файл и возвращает массив


#### 3. `collect_all_arrays()`
Собирает все массивы из тестовых файлов.

**Алгоритм:**
- Создает массив из трех указателей
- Вызывает read() для каждого тестового файла:
  - test_sorted.txt
  - test_reverse.txt
  - test_random.txt
- Возвращает массив массивов


#### 4. `shell_sort(Data** array, int size)`
Сортирует массив структур методом Шелла по ключу.

**Алгоритм:**
- Начинает с шага h = size/2
- Для каждого шага:
  - Сортирует элементы, отстоящие на h позиций
  - Использует алгоритм вставками для каждого подмассива
- Уменьшает шаг (h = h/2)
- Повторяет до h = 0


#### 5. `binary_search(Data** array, int count, double search_key)`
Выполняет бинарный поиск элемента по ключу.

**Алгоритм:**
- Устанавливает границы: left = 0, right = count - 1
- Пока left ≤ right:
  - Вычисляет середину: mid = (left + right) / 2
  - Если ключ совпадает — возвращает элемент
  - Если искомый ключ меньше — right = mid - 1
  - Если искомый ключ больше — left = mid + 1
- Если элемент не найден — возвращает NULL


#### 6. `sort_all_data(Data*** data_arrs, int* count_arrs)`
Сортирует все массивы и выводит результаты.

**Алгоритм:**
- Для каждого из трех массивов:
  - Выводит исходный массив
  - Вызывает shell_sort()
  - Выводит отсортированный массив


#### 7. `show(Data** main_massive, int count)`
Выводит содержимое массива на экран.

**Алгоритм:**
- Проходит по всем элементам массива
- Для каждого элемента выводит:
  - Номер элемента
  - Значение ключа
  - Строковые данные


#### 8. `read_count(char* path)`
Читает количество элементов из файла.

**Алгоритм:**
- Открывает файл
- Читает первую строку
- Преобразует строку в число
- Закрывает файл и возвращает результат


#### 9. `read_count_in_arr()`
Читает количество элементов из всех файлов.

**Алгоритм:**
- Создает массив из трех элементов
- Вызывает read_count() для каждого файла
- Возвращает массив с количествами

## Особенности реализации

- Все динамически выделенные структуры освобождаются в тестах

- Используется `free()` для освобождения памяти

- Используется `fopen()/fclose()` для работы с файлами
- Проверка успешности открытия файлов
- Чтение данных построчно с помощью `fgets()`
