## Описание реализации

### Структуры данных

#### 1. Узел дерева выражений

```c
typedef enum {
    NODE_NUMBER,     // Числовой узел
    NODE_OPERATOR,   // Узел с оператором
    NODE_VARIABLE    // Узел с переменной
} NodeType;

typedef struct node {
    NodeType type;
    union {
        int number;      // Для числовых узлов
        char op;         // Для операторов
        char variable;   // Для переменных
    } value;
    struct node* left;   // Левый операнд
    struct node* right;  // Правый операнд
    int depth;           // Глубина узла
} Node;
```

**Особенности:**
- Используется типизированная структура с enum для различения типов узлов
- Union позволяет эффективно использовать память
- Избегает путаницы между ASCII-кодами символов и числовыми значениями

#### 2. Стек для операторов

```c
typedef struct stack_ptr {
    char data;
    struct stack_ptr* next;
} stack_ptr;
```

Стек реализован в виде односвязного списка с операциями:
- `push()` — добавление элемента
- `pop()` — извлечение элемента
- `top()` — просмотр вершины без извлечения
- `stack_init()` — инициализация со значением-сторожем `'@'`

### Реализованные модули

#### 1. `validation.h` — Валидация выражений

**Функция:** `int validation(char* str)`

**Алгоритм:**
- Использует конечный автомат с флагом `expecting_operand`
- Проверяет чередование операндов и операторов
- Подсчитывает баланс скобок
- Поддерживает многозначные числа, переменные (буквы) и пробелы

**Проверки:**
- Корректная последовательность токенов
- Сбалансированность скобок
- Отсутствие двух операторов/операндов подряд
- Выражение начинается с операнда или `(`
- Выражение заканчивается операндом или `)`

#### 2. `stack.h` — Стек операторов

**Функции:**
- `stack_init()` — инициализация стека
- `push(char data, stack_ptr** head)` — добавление в стек
- `pop(stack_ptr** head)` — извлечение из стека
- `top(stack_ptr* head)` — просмотр вершины
- `show(stack_ptr* head)` — вывод содержимого (рекурсивно)


#### 3. `check_priority.h` — Приоритеты операторов

**Функция:** `int priority_check(char oper)`

**Возвращаемые значения:**
- `1` для `+`, `-`
- `2` для `*`, `/`
- `3` для `^`


#### 4. `rpn.h` — Преобразование в RPN

**Функция:** `char* rpn(char* str)`

**Алгоритм (Shunting Yard):**
1. Инициализация стека операторов и выходной строки
2. Для каждого символа:
   - **Пробел** — пропускаем
   - **Цифра** — считываем все число, добавляем в выход с пробелом
   - **Буква** — считываем переменную, добавляем в выход с пробелом
   - **Открывающая скобка** — push в стек
   - **Закрывающая скобка** — pop до открывающей скобки
   - **Оператор**:
     - Выталкиваем операторы с большим приоритетом
     - Для операторов с равным приоритетом: выталкиваем если не `^` (правоассоциативность)
     - Push текущего оператора
3. Выталкиваем оставшиеся операторы из стека

**Особенность:** Использует `(unsigned char)` для безопасной работы с `ctype.h` функциями

**Пример:**
```
Вход:  "(123+456-1)*b*1"
Выход: "123 456 + 1 - b * 1 *"
```

#### 5. `rpn_to_tree.h` — Построение дерева из RPN

**Функция:** `Node* rpn_to_tree(char* str_rpn)`

**Алгоритм:**
1. Создаем стек узлов дерева
2. Токенизируем RPN-строку (разделитель — пробел)
3. Для каждого токена:
   - **Число** → создаем NODE_NUMBER, push в стек
   - **Переменная** → создаем NODE_VARIABLE, push в стек
   - **Оператор** → pop два операнда, создаем NODE_OPERATOR с ними как потомками, push результат
4. Возвращаем вершину стека (корень дерева)


#### 6. `tree.h` — Операции с деревом

**Функции:**

1. **`initiate_tree(int head_data)`** — создание узла
   - Автоматически определяет тип узла по значению
   - ASCII-коды операторов → NODE_OPERATOR
   - ASCII-коды букв → NODE_VARIABLE
   - Остальное → NODE_NUMBER

2. **`print_tree(Node* head)`** — визуализация дерева
   - Рекурсивный обход (правый-корень-левый)
   - Отступы показывают уровень вложенности
   - Switch по NodeType для корректного отображения

3. **`remove_unit_factors_tree(Node* node)`** — удаление умножения на 1
   - Post-order обход (сначала обрабатываем потомков)
   - Если узел — оператор `*` и один из операндов — лист с числом `1`:
     - Удаляем узел и листовой `1`
     - Возвращаем второй операнд
   - Иначе возвращаем узел без изменений

4. **`is_leaf(Node* node)`** — проверка, является ли узел листом
5. **`is_operator(Node* node)`** — проверка типа узла

#### 7. `calculate_tree.h` — Вычисление выражений

**Функция:** `Node* simplify(Node* top)`

**Алгоритм:**
1. Post-order обход дерева
2. Рекурсивно упрощаем левое и правое поддеревья
3. Если узел — оператор и оба операнда — числа:
   - Вычисляем операцию
   - Преобразуем узел-оператор в NODE_NUMBER с результатом
   - Освобождаем память потомков
4. Возвращаем упрощенный узел

**Поддерживаемые операции:** `+`, `-`, `*`, `/`, `^`

**Особенность:** Частичное вычисление — если в выражении есть переменные, вычисляются только константные подвыражения.

**Вспомогательные функции:**
- `is_variable(Node* node)` — проверка, является ли узел переменной
- `is_number_node(Node* node)` — проверка, является ли узел числом

## Тестирование

Реализовано 70+ модульных тестов с использованием стандартной библиотеки `assert.h`, разбитых на 6 категорий:

### 1. Тесты валидации (test_validation)
- Простые выражения: `2+3`
- Выражения со скобками: `(2+3)*4`
- Многозначные числа: `123+456`
- Переменные: `a+b*c`, `x1+y2`
- Выражения с пробелами: `2 + 3`
- Невалидные выражения:
  - Два оператора подряд: `2++3`
  - Незавершенное выражение: `2+`
  - Начало с оператора: `+2`
  - Несбалансированные скобки: `(2+3`, `2+3)`
  - Два операнда подряд: `2 3`

### 2. Тесты RPN (test_rpn_conversion)
- Простое сложение: `2+3` → `2 3 +`
- Приоритет операторов: `2+3*4` → `2 3 4 * +`
- Скобки: `(2+3)*4` → `2 3 + 4 *`
- Многозначные числа: `123+456` → `123 456 +`
- Левая ассоциативность: `2-3-4` → `2 3 - 4 -`
- Правая ассоциативность степени: `2^3^4` → `2 3 4 ^ ^`
- Переменные: `a+b*c` → `a b c * +`

### 3. Тесты построения дерева (test_tree_building)
- Построение простого дерева из `2+3`
- Проверка типов узлов (NODE_OPERATOR, NODE_NUMBER)
- Проверка значений в узлах
- Многозначные числа в дереве: `123+456`
- Переменные в дереве: `a+b`
- Проверка структуры (левый/правый потомки)

### 4. Тесты упрощения (test_simplification)
- Удаление 1 справа: `5*1` → `5`
- Удаление 1 слева: `1*5` → `5`
- Сложное выражение: `(2+3)*1` → `(2+3)`
- Несколько единиц: `5*1*1` → `5`
- 1 не при умножении: `5+1` (не изменяется)

### 5. Тесты вычисления (test_calculation)
- Простое сложение: `2+3` → `5`
- Вычитание: `10-3` → `7`
- Умножение: `4*5` → `20`
- Деление: `15/3` → `5`
- Степень: `2^3` → `8`
- Сложное выражение: `(2+3)*4` → `20`
- С переменными: `2+a` (оператор остается)
- Частичное вычисление: `(2+3)*a` → `5*a`

### 6. Комплексные тесты (test_complex)
- Полный pipeline: валидация → RPN → дерево → упрощение → вычисление
- Тест 1: `(123+456-1)*20*1` → `11560`
- Тест 2: `(10+5)*b*1` → упрощение с переменной → `15*b`

## Сборка проекта

### Makefile

```makefile
CC = gcc
CFLAGS = -Wall -Wextra
LDFLAGS = -lm

SRC_DIR = src
TESTS_DIR = tests

HEADERS = $(SRC_DIR)/*.h

main: $(SRC_DIR)/main.c $(HEADERS)
	$(CC) $(CFLAGS) $(SRC_DIR)/main.c -o $(SRC_DIR)/main $(LDFLAGS)

tests: $(TESTS_DIR)/tests.c $(HEADERS)
	$(CC) $(CFLAGS) $(TESTS_DIR)/tests.c -o $(TESTS_DIR)/tests $(LDFLAGS)

run: main
	./$(SRC_DIR)/main

run_tests: tests
	./$(TESTS_DIR)/tests

clean:
	rm -f $(SRC_DIR)/main $(TESTS_DIR)/tests

all: main tests

.PHONY: run run_tests clean all
```

### Команды сборки

```bash
make all        # Собрать main и tests
make main       # Собрать только основную программу
make tests      # Собрать только тесты
make run        # Запустить основную программу
make run_tests  # Запустить тесты
make clean      # Удалить скомпилированные файлы
```

### Особенности компиляции

- **Флаги:** `-Wall -Wextra` для строгой проверки
- **Линковка:** `-lm` для подключения математической библиотеки (функция `pow()`)
- **Пути:** используются переменные `SRC_DIR` и `TESTS_DIR` для организации структуры
- **.PHONY:** объявлены служебные цели для избежания конфликтов с файлами
