### Модуль `matrix.h` — матрица смежности

#### `initiate_matrix(int size)` → `int**`
Выделяет память под квадратную матрицу `size × size`. Каждая строка — отдельный массив `int`.

#### `write_matrix_from_terminal(int size)` → `int**`
Считывает матрицу с терминала через `scanf`. Реализована валидация: при вводе отрицательного числа запрашивает повтор ввода.

#### `take_element(int str, int column, int** matrix)` → `int`
Возвращает элемент матрицы по 1-индексированным строке и столбцу.

#### `print_matrix(int size, int** matrix)`
Выводит матрицу в консоль построчно.

#### `free_matrix(int** matrix, int size)`
Освобождает память: сначала каждую строку `matrix[i]`, затем массив указателей `matrix`.

### Модуль `dijkstra.h` — алгоритм Дейкстры

#### `check_if_in_checked(int peak, int* checked, int size)` → `int`
Проверяет, содержится ли вершина `peak` в массиве посещённых вершин. Возвращает 1 если да, 0 если нет.

#### `min_num(int* out, int* checked, int size)` → `int`
Находит индекс непосещённой вершины с минимальным расстоянием в массиве `out`. Возвращает -1, если все достижимые вершины посещены (расстояния равны `INT_MAX`).

#### `dijkstra_algorithm(int** matrix, int size)` → `int*`
Основная функция. Принимает матрицу смежности и число вершин. Возвращает массив кратчайших расстояний от вершины 0 до всех остальных. Недостижимые вершины имеют значение `INT_MAX`.

**Алгоритм:**
1. Инициализация: `out[0] = 0`, все остальные `out[i] = INT_MAX`
2. Массив `checked` инициализируется `-1` через `memset`
3. Цикл: обновление расстояний до соседей текущей вершины, выбор следующей вершины через `min_num`
4. Завершение при `min_num == -1`

### Модуль `benchmark.c` — бенчмаркинг

Генерирует случайные ориентированные графы разных размеров и замеряет время работы алгоритма. Каждый тест запускается 5 раз, берётся среднее значение.

#### Результаты бенчмаркинга

| Размер графа (V) | Время (мс) |
|---|---|
| 10 | 0.004 |
| 50 | 0.072 |
| 100 | 0.372 |
| 250 | 4.347 |
| 500 | 16.273 |
| 1000 | 100.749 |

Результаты подтверждают теоретическую сложность $O(V^2)$: при удвоении числа вершин время растёт примерно в 4 раза.

## Тестирование

Тесты написаны с использованием фреймворка **Google Test** и скомпилированы с `extern "C"` для подключения C-заголовков. Для подмены пользовательского ввода используется `mock_stdin` — функция, записывающая тестовые данные во временный файл и перенаправляющая на него `stdin` через `freopen`.

### Группы тестов

#### `Matrix` — 3 теста

| Тест | Что проверяет |
|------|--------------|
| `AllValidValues` | Корректный ввод 2×2 матрицы, все элементы записаны верно |
| `NegativeValue` | При вводе отрицательного значения запрашивается повтор, записывается корректное |
| `TakeElement` | `take_element` возвращает правильный элемент по 1-индексированным координатам |

#### `Dijkstra` — 5 тестов

| Тест | Что проверяет |
|------|--------------|
| `SingleVertex` | Граф из одной вершины: расстояние до себя = 0 |
| `TwoVertices` | Прямое ребро 0->1 с весом 5: `out[1] == 5` |
| `UnreachableVertex` | Недостижимая вершина остаётся с расстоянием `INT_MAX` |
| `ShortestPathThroughIntermediate` | Путь 0->2->1 (длина 5) предпочтительнее прямого 0->1 (длина 10) |
| `LinearChain` | Цепочка 0->1->2->3 с весами 1,2,4: расстояния 0,1,3,7 |

## Сборка и запуск проекта через CMake c использованием Makefile

Проект полностью собирается через CMake с автоматическим скачиванием GoogleTest (FetchContent). Все этапы (тесты, main, benchmark, построение графика) выполняются последовательно одной командой.

### Основные команды

```bash
# Генерация сборочных файлов
cmake -S . -B build
# Сборка и запуск всех этапов
cmake --build build --target full_pipeline
```

Или через Makefile:

```bash
make run
```

### Особенности
- Пути к файлам скорректированы: benchmark.txt читается/пишется по пути ../DATA/benchmark.txt относительно build.
- GoogleTest подключается автоматически, вручную ничего ставить не нужно.
- Все этапы (тесты → main → benchmark → Python-скрипт) запускаются последовательно.
