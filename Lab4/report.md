## Теоретическая часть

**АВЛ-дерево** — самобалансирующееся бинарное дерево поиска, в котором для каждого узла высоты левого и правого поддеревьев отличаются не более чем на 1. Баланс поддерживается автоматически через ротации при вставке и удалении.

**Баланс-фактор** узла:

$$bf(v) = h(\text{left}) - h(\text{right})$$

Допустимые значения: $bf \in \{-1, 0, 1\}$.

При нарушении выполняются ротации:
- **Правая ротация** — при $bf > 1$ и левый потомок не перегружен вправо
- **Левая ротация** — при $bf < -1$ и правый потомок не перегружен влево
- **Лево-правая** — при $bf > 1$ и левый потомок перегружен вправо
- **Право-левая** — при $bf < -1$ и правый потомок перегружен влево


## Описание реализации

### Структура данных

```c
typedef struct Node {
    char key[6];       // Ключ — строка до 5 символов
    double data;       // Вещественное значение
    struct Node* left;
    struct Node* right;
    int height;        // Высота поддерева для балансировки
} Node;
```

**Особенности:**
- Ключ — строка длиной от 1 до 6 символов, только латинские буквы
- Значение — `double`, хранится `NAN` при отсутствии элемента
- Поле `height` обновляется после каждой операции изменения дерева

---

### Модуль `tree.h` — АВЛ-дерево

#### `height_count(Node* head)` → `int`
Рекурсивно вычисляет высоту поддерева. Возвращает 0 для `NULL`.

#### `balance_factor(Node* n)` → `int`
Вычисляет баланс-фактор узла как разницу высот левого и правого поддеревьев.

#### `right_rotate(Node* n)` / `left_rotate(Node* n)` → `Node*`
Выполняют простые ротации вокруг узла. Обновляют высоты затронутых узлов.

#### `avl_insert(char* key, double data, Node* head)` → `Node*`
Рекурсивно вставляет новый узел с заданным ключом и значением. После вставки пересчитывает высоту и при необходимости выполняет одну из четырёх ротаций. Дубликаты игнорируются.

**Алгоритм:**
1. Стандартная рекурсивная вставка по `strcmp`
2. Обновление высоты текущего узла
3. Вычисление баланс-фактора
4. При нарушении баланса — ротация соответствующего типа

#### `avl_delete(Node* head, char* key)` → `Node*`
Рекурсивно удаляет узел. При удалении узла с двумя потомками заменяет его минимальным элементом правого поддерева. После удаления восстанавливает баланс.

#### `search(char* key, Node* head)` → `double`
Рекурсивный поиск по ключу. Возвращает значение узла или `NAN`, если ключ не найден или дерево пустое.

> Проверка результата через `isnan()`, а не `== NAN` — NaN не равен сам себе по стандарту IEEE 754.

#### `print_tree(Node* head, FILE* fp)`
Визуализирует дерево обходом правое → корень → левое с отступами. Выводит в консоль, а при `fp != NULL` — одновременно в файл.

### Модуль `read.h` — обработка команд

#### Функции валидации

| Функция | Описание | Коды ошибок |
|---------|----------|-------------|
| `validate_cmd(int cmd)` | Допустимы команды 1–4 | 0 = ОК, 1 = ошибка |
| `validate_key(char* key, int len)` | Только латинские буквы, 1–6 символов | 1 = пусто, 2 = слишком длинный, 3 = недопустимые символы |
| `validate_value(char* val, int len)` | Числовой формат с опциональным знаком и точкой | 1 = пусто, 2 = неверный формат |

#### `read(int output)` → `Node*`

Главная функция модуля. Читает файл `../DATA/test.txt` построчно и выполняет команды:

| Команда | Формат строки | Действие |
|---------|--------------|----------|
| `1 key value` | `1 One 3.14` | Вставка элемента |
| `2 key` | `2 One` | Удаление элемента |
| `3 key` | `3 One` | Поиск элемента |
| `4` | `4` | Вывод дерева |

При `output == 1` все результаты дополнительно записываются в `../DATA/test_out.txt` (режим дозаписи `"a"`).

### Модуль `main.c`

```c
int main() {
    Node* root = read(1);
    if (root == NULL) {
        printf("Дерево пустое или файл не найден\n");
        return 1;
    }
}
```

## Тестирование

Тесты написаны с использованием фреймворка **Google Test** и скомпилированы с `extern "C"` для подключения C-заголовков.

### Группы тестов

#### `AVLInsert` — 4 теста
| Тест | Что проверяет |
|------|--------------|
| `InsertSingleNode` | Корень создаётся с верными key и data |
| `InsertMultipleNodes` | Все вставленные элементы находятся поиском |
| `InsertMaintainsAVLProperty` | После 5 вставок баланс-фактор каждого узла в $\{-1, 0, 1\}$ |
| `InsertDuplicateKeyIgnored` | Повторная вставка не меняет значение |

#### `Search` — 4 теста
| Тест | Что проверяет |
|------|--------------|
| `FindExistingKey` | Возвращает точное значение `double` |
| `FindNonExistingKey` | Возвращает `NAN` |
| `SearchInEmptyTree` | `search(..., NULL)` возвращает `NAN` |
| `SearchEmptyKey` | Пустой ключ возвращает `NAN` |

#### `AVLDelete` — 5 тестов
| Тест | Что проверяет |
|------|--------------|
| `DeleteLeafNode` | Лист удаляется, остальные узлы на месте |
| `DeleteNodeWithTwoChildren` | Узел с двумя потомками удаляется корректно |
| `DeleteMaintainsAVLProperty` | После удалений дерево остаётся сбалансированным |
| `DeleteNonExistingKey` | Дерево не меняется при удалении отсутствующего ключа |
| `DeleteFromEmptyTree` | `avl_delete(NULL, ...)` возвращает `NULL` |

#### `Height` — 3 теста
| Тест | Что проверяет |
|------|--------------|
| `EmptyTree` | `height_count(NULL) == 0` |
| `SingleNode` | Высота одного узла равна 1 |
| `BalancedTree` | Высота сбалансированного дерева из 3 узлов равна 2 |

#### `Balance` — 1 тест
| Тест | Что проверяет |
|------|--------------|
| `RootBalanced` | `balance_factor` корня при {A, B, C} равен 0 |
